RTPI REST Web Services API

RTPI REST Web Services API (RRWS API) provides REST interface to retrieve information on real time bus information, timetables and bus stops.

Each method available in RRWS API will be invoked using plain HTTP Request (GET method). Depending on parameters provided, result will be returned either in XML or JSON format (if format is not provided then JSON is used by default). Response will contain requested information (see method specification for details) and error status of the request.

Error Codes

Error CodeDescription
0Success
1No Results
2Missing Parameter
3Invalid Parameter
4Sceduled Downtime
5Unexpected System Error

Error code is included in response for each message. For some error codes additional details will be provided in free text format (Error Message field).

get /busstopinformation
Endpoint to retrieve bus stop information (getBusStops)

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json
  • application/xml

Query parameters

format (optional)
Query Parameter — If not provided JSON format will be used by default
operator (optional)
Query Parameter — If provided retrieves routes for the given operator
stopid (optional)
Query Parameter — The bus stop id, if not provided all bus stops are returned
stopname (optional)
Query Parameter — The stop name, only matching stops are returned

Return type

BusStopList

Example data

Content-Type: application/json
{
  "errormessage" : "aeiou",
  "numberofresults" : 5,
  "results" : [ {
    "fullnamelocalized" : "aeiou",
    "operators" : [ {
      "routes" : [ "aeiou" ],
      "name" : "ir"
    } ],
    "latitude" : "53.34642974",
    "stopid" : "HSTON",
    "fullname" : "aeiou",
    "shortnamelocalized" : "aeiou",
    "shortname" : "aeiou",
    "displaystopid" : "Dublin Heuston Station",
    "longitude" : "-6.294607473"
  } ],
  "errorcode" : "0",
  "timestamp" : "17/06/2017 11:44:35"
}

Example data

Content-Type: application/xml

  0
  string
  5
  17/06/2017 11:44:35

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json
  • application/xml

Responses

200
